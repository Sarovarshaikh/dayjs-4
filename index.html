Q1. What is a Temporal Dead Zone?

Ans- A temporal dead zone (TDZ) is the area of a block where a variable is inaccessible until the moment the computer completely initializes it with a value.

Q2. What is the for-in loop in JavaScript? Give its syntax.

Ans- The for...in statements combo iterates (loops) over the properties of an object.The code block inside the loop is executed once for each property.Keys of an object or strings can be accessed by for-in loop.

        let object = {
            fname: "Sarovar",
            lname: "Shaikh",
        }
        
        for(let j in object){
            console.log(j); // fname, lname
        }
        
        let array = ["ana","bana","cana"];
        
        for(let i in array){
            console.log(i);
        }

Output- 0, 1, 2

Q3. Explain Local Scope, Block Scope, Functional Scope and Scope Chain in javascript.

Ans- Scope in JS

  • Local Scope- accessible inside that function only
  • Block Scope- Inside block of code, it will access the variables inside that block only.
  • Functional Scope- Variables are valid only inside that function.
  • Scope Chain- Variables are accessible inside and outside their scope.

Q6. Write code to explain map and filter in arrays.

Ans- May and filter in arrays:

When to use map: If we want to perform the same operation/transformation on each element of the array and get back a new array of the same length with the transformed values.

        const arrayOne = [100, 200, 300, 400, 500];
        
        const arrayTwo = arrayOne.map( x => x + 50);

Output- [150,250,350,450,550]

When to use filter- Every element of the array is passed to the callback function. On each iteration, if the callback returns true, then that element will be added to the new array, otherwise, it is not added to the new array.

 const arrayThree = arrayOne.filter( x => x > 300);
        console.log(arrayThree);

Output- [400,500]

Q7. Explain passed by value and passed by reference.

Ans- Passed by value- works with Primitive Datatypes(null,undefined,boolean,number,bigint,string,symbol)

In Pass by value, parameters passed as an arguments create its own copy. So any changes made inside the function is made to the copied value not to the original value.

        let a = 10;
        let b = 5;
        
        b = a + 5;
        console.log(b);
        console.log(a);

Output-

15

10

Passed by Reference- works with Primitive Datatypes( arrays and objects )

In Pass by reference, parameters passed as an arguments does not create its own copy, it refers to the original value so changes made inside function affect the original value.

        let objectt = {
            name: "Don",
            age: 24,
        }
        
        let objectt1 = objectt;
        
        objectt1.age = 28,
        console.log(objectt1);

Output- name:Don, age:28

Q8. Please explain Self Invoking Function and its code.

In JavaScript, a “Self-Invoking” function is a type of function that is invoked or called automatically after its definition. The execution of such an anonymous function is done by enclosing it in a parenthesis set followed by another set of parenthesis.

Ans- It is also called ad Immediately Invoke Function Expression.i.e, IIFE function

It is a way to immediately call a function

It needs to be called as soon as it is defined.

Function expressions can be made "self-invoking".

Syntax-

        (function () {
            console.log("Execute me first !!");
          })();

Output- Execute me first !!
